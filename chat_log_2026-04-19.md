## 三、新增架構元素說明

### Critic Agent 委員會（Crew 4 防詐審議）

流程：
```
Crew 1+2+3 並行產出（5P + XGBoost + 法規）
       ↓
Crew 4 Critic Agent 委員會
  委員 A（風控）：合規閾值是否超過？
  委員 B（業務）：還款能力是否充分？
  委員 C（法規）：法規紅線是否符合？
       ↓
投票：核准 / 有條件核准 / 婉拒 / REWORK
       ↓
Admin 後台行員 HITL 最終決策
```

升級路由三級：
- Level 1（防詐分數 ≤ 40）→ 一鍵確認
- Level 2（41–70）→ 資深行員審查
- Level 3（> 70）→ 凍結 + 主管強制介入

### 資料策略（S3.5 知識金庫）

MD + YAML 結構化資料的三重價值：
1. RAG 即時可用（新增法規 30 秒進知識庫）
2. ML 訓練語料（每筆批覆書 = 一筆監督學習標籤）
3. 跨 LLM 通用，不綁廠商（Azure OpenAI / Qwen / Gemini 均可讀）

### 推薦引擎三層架構（完整版）

- Layer 1：Content-Based Filtering（isEligible() 資格過濾）
- Layer 2：Collaborative Filtering（生命週期套餐：首購族/退休族/軍公教）
- Layer 3：LLM Instruction Following（Qwen2.5:14b 個人化說明）
  學術依據：InstructRec（ArXiv 2023）
