## 三、確認的黑客松架構敘事

### 主框架：Harness Engineering（Martin Fowler）

> Agent = Model + Harness
> Model 決定 AI 有多聰明，Harness 決定 AI 有多可靠。

```
【護欄層】Harness
  Guides（前饋）：CLAUDE.md · 對話狀態機 · 合規紅線（DBR/LTV）
  Sensors（反饋）：軍機處監控看板 · Azure App Insights（未來）

【編排層】Co-Pilot 指揮艙（Planner / DAG）
  workflowService.ts — Phase 1 → 2 → 3

【執行層】六大 Pilot Crew（Manager + Worker）
  Crew 1 徵信 / Crew 2 鑑估 / Crew 3 法規
  Crew 4 審議 / Crew 5 文件 / Crew 6 客服

【底座層】Azure 雲端服務
```

### 輔助框架：Hierarchical Agent Architecture（三層）

| 層級 | 個金 Co-Pilot 對應 |
|------|-----------------|
| Planner（策略調度）| Co-Pilot 指揮艙 `workflowService.ts` |
| Manager（邏輯管理）| 六大 Pilot Crew 機長 |
| Worker（原子執行）| 各 Crew 內單一職責執行單元 |
| Critic Agent | Crew 4 三委員交叉驗證 |

## 四、推薦系統演算法研討與 LLM 混合架構規劃

### 背景

討論如何將現有規則式推薦引擎包裝成 LLM 混合推薦系統，
對應黑客松 Demo 的技術敘事，並參考學術論文與 Netflix 演算法進行架構升級規劃。

---

### 推薦系統四大演算法概覽

| 類別 | 代表技術 | 說明 |
|------|---------|------|
| **協同過濾 CF** | User-based / Item-based | 找相似用戶或相似商品進行推薦 |
| **矩陣分解 MF** | SVD、ALS | 分解用戶-商品矩陣預測評分（Netflix Prize 核心）|
| **內容過濾** | Content-Based Filtering | 依商品屬性標籤媒合用戶偏好 |
| **混合式** | Hybrid Systems | 結合上述，解決冷啟動問題 |
| **LLM 推薦** | InstructRec、Zero-shot | LLM 理解自然語言意圖進行推薦（2023 最新）|

---

### Netflix 演算法演進

| 世代 | 技術 | 特點 |
|------|------|------|
| 第一代（2006-2009）| 矩陣分解（SVD）| Netflix Prize 催生，預測未看過的評分 |
| 第二代（2016後）| Wide & Deep + Embedding | 記憶歷史特徵（Wide）+ 泛化新特徵（Deep）|
| 現在 | 多訊號疊加 + Learning to Rank | 觀看史 + 相似用戶 + 標籤 + 情境（時段/裝置）|
| 延伸 | Contextual Bandit | 同一部電影對不同用戶顯示不同縮圖 |

**Netflix 冷啟動解法**：
1. 大量標籤員手動標記數千個內容屬性
2. Explore & Exploit：新用戶先看首頁 Top 10，觀察第一次點擊破冰

---

### InstructRec 論文核心（ArXiv 2023）

**論文名稱**：*Recommendation as Instruction Following: A Zero-shot Learning Approach*

| 特性 | 說明 |
|------|------|
| 核心概念 | 將推薦視為「指令遵循」語言任務 |
| 冷啟動 | Zero-shot：不需要行為紀錄，憑自然語言描述即可推薦 |
| 優勢 | 可解釋性高，能用語言說明推薦理由 |
| 對比 Netflix | Netflix 是被動捕捉隱性偏好；InstructRec 主動理解顯性意圖 |

---

### 冷啟動解法五大策略（學術整理）

| 策略 | 說明 | 個金 Co-Pilot 對應 |
|------|------|-----------------|
| 內容特徵關聯 | 用戶靜態屬性（年齡/職業/地區）| `isEligible()` 資格篩選 |
| 啟發式調查（Onboarding）| 首次登入讓用戶選標籤 | 申請表單填表過程 |
| 熱門推薦 | Top-K 熱門商品先推 | `priorityOrder` 優先序 |
| 跨域推薦 | 引用其他平台行為資料 | — |
| LLM Zero-shot | 憑少量描述進行語義推理 | Qwen2.5 說明生成（規劃中）|

**關鍵洞察**：銀行貸款申請必須填齊欄位，冷啟動問題**天然不存在**。
比 Netflix 的冷啟動挑戰更容易解決，這是金融推薦的架構優勢。

### 個金 Co-Pilot — Netflix × InstructRec 混合架構

#### 現況診斷

| 組件 | 現在 | 學術分類 |
|------|------|---------|
| `isEligible()` | 規則篩選 | Content-Based Filtering |
| `priorityOrder` | 固定優先序 | Popularity Ranking |
| 申請表單 | 必填欄位 | Structured Instruction（InstructRec）|
| `buildReasons()` | 模板字串 | ❌ 非 AI，待升級 |
| `crossSell` | 靜態硬綁定 | ❌ 非 CF，待升級 |

#### 目標三層混合架構

```
Layer 1：Content-Based（Netflix 冷啟動策略）
  用戶屬性（年齡/職業/收入）→ isEligible() 資格篩選 → 候選商品池

Layer 2：Collaborative Filtering（Netflix 核心）
  「申辦青安的客戶，68% 同時辦了御璽卡 + 房貸壽險」
  → crossSell 從靜態升級為 CF 驅動的套餐推薦

Layer 3：LLM Instruction Following + Explanation（InstructRec）
  申請表單 = Structured Instruction
  Qwen2.5 → 根據客戶條件生成個人化套餐說明
```

#### 生命週期套餐推薦示範

| 客群 | 主推貸款 | CF 推薦配套 | LLM 說明角度 |
|------|---------|-----------|------------|
| 首購族（30歲/公務員）| 青安房貸 | 御璽卡 + 房貸壽險 + CoBaby帳戶 | 未來30年節省試算 |
| 退休族（62歲/自有房）| 以房養老 | 長照保險 + 安養信託 + 黃金存摺 | 每月穩定現金流規劃 |
