## 三、確認的黑客松架構敘事

### 主框架：Harness Engineering（Martin Fowler）

> Agent = Model + Harness
> Model 決定 AI 有多聰明，Harness 決定 AI 有多可靠。

```
【護欄層】Harness
  Guides（前饋）：CLAUDE.md · 對話狀態機 · 合規紅線（DBR/LTV）
  Sensors（反饋）：軍機處監控看板 · Azure App Insights（未來）

【編排層】Co-Pilot 指揮艙（Planner / DAG）
  workflowService.ts — Phase 1 → 2 → 3

【執行層】六大 Pilot Crew（Manager + Worker）
  Crew 1 徵信 / Crew 2 鑑估 / Crew 3 法規
  Crew 4 審議 / Crew 5 文件 / Crew 6 客服

【底座層】Azure 雲端服務
```

### 輔助框架：Hierarchical Agent Architecture（三層）

| 層級 | 個金 Co-Pilot 對應 |
|------|-----------------|
| Planner（策略調度）| Co-Pilot 指揮艙 `workflowService.ts` |
| Manager（邏輯管理）| 六大 Pilot Crew 機長 |
| Worker（原子執行）| 各 Crew 內單一職責執行單元 |
| Critic Agent | Crew 4 三委員交叉驗證 |
