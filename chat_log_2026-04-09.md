## 一、Harness Engineering 框架評估

### 來源

Martin Fowler 文章：[Harness Engineering for Coding Agent Users](https://martinfowler.com/articles/harness-engineering.html)
搭配 Wilson Huang FB 貼文：Claude Managed Agents 發布、Harness 管理各種 Agent 痛點討論。

### 核心概念

**Agent = Model + Harness**

| 概念 | 說明 |
|------|------|
| Guides（前饋）| 事前引導 Agent 行為，限縮輸出空間 |
| Sensors（反饋）| 事後監控 Agent 輸出，觸發自我修正 |
| Computational Sensors | 決定性、毫秒級（測試、Linter、閾值計算）|
| Inferential Sensors | 語義、非決定性（AI 程式碼審查、委員會交叉驗證）|

**白話解釋：Harness Engineering 是「給 AI 裝護欄的工程學」。**

### 我們已有的 Harness

| 已有實作 | Harness 術語 |
|---------|-------------|
| `CLAUDE.md` | Guide |
| `conversationStateMachine.ts` | Guide（限制合法狀態）|
| `recordAgentCall()` | Sensor（計算型）|
| `agentMonitorStore.ts` | Sensor（即時狀態）|
| DBR ≤ 22x / LTV ≤ 85% | Guide（憲法紅線）|

## 二、Azure AI Foundry Agent Service 評估

### 三大方案

| 方案 | 說明 | 對應我們的架構 |
|------|------|--------------|
| Foundry Agent Service | 完全托管 runtime + scaling | 六大 Pilot Crew 未來部署 |
| Hosted Agents（Preview）| MAF/LangGraph/CrewAI 直接部署，不需 containerization | 現有 MAF Crew 可直接上雲 |
| Foundry Memory | 跨 session 長期記憶，不需自建 embedding DB | 取代 In-Memory 狀態 |

### 企業安全特性

- Microsoft Entra identity：每個 Agent 有專屬身份
- Private Networking：資料落地合規
- 細粒度 RBAC：Agent 層級權限控制

### 限制

Hosted Agents 截至 2026/03 限定 North Central US 區域，台灣資料落地需注意。

### 關鍵加分點

> Azure 是目前唯一同時托管 Anthropic Claude 和 OpenAI GPT 的雲端平台。
> 4/22 簡報強調「Foundry 統一管理 Claude + GPT 混合架構」是強力亮點。

## 三、Agent Governance 框架評估

### 來源

[bounce12340/agent-governance](https://github.com/bounce12340/agent-governance)

### 核心設計

五角色模型：立法（需求→法律）/ 行政（實作）/ 司法（驗證）/ Harness（證據把關）/ 憲法（全域規則）

**判決五種狀態**：PASSED / INCOMPLETE / REWORK / REJECTED / 補件請求

**核心原則**：「只有當法律 + 證據 + 產出三者一致，才 PASSED」

### 對我們的借鑑價值

| Agent Governance 概念 | 個金 Co-Pilot 對應 | 缺口 |
|---------------------|-----------------|------|
| Constitution（憲法）| DBR/LTV 金融紅線 | ✅ 已有 |
| Executive（行政）| 六大 Pilot Crew | ✅ 已有 |
| Judiciary（司法）| 委員會三委員審議 | ⚠️ 缺 REWORK 狀態 |
| Harness（證據把關）| agentMonitorStore | ⚠️ 有監控，缺「證據包」|

### 具體可借鑑（未來實作）

1. 委員會加入 `rework`（待補件）狀態
2. 審核附上「證據包」：5P 評分 + 防詐分數 + 鑑價信心區間
