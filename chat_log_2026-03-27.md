## 二、論文核心內容摘要

### 論文資訊

- **標題**：預測台灣房地產市場趨勢之模型——應用深度學習技術
- **作者**：蔡繡容、夏政瑋（國立高雄科技大學）
- **出版**：住宅學報，第32卷第2期，2023年12月

### 模型架構（Zaremba et al. 2014 LSTM）

```
輸入：shape(batch, 4, 25)  ← 4 季落後期 × 25 特徵變數
  ↓
LSTM Layer 1（N1=64, return_sequences=True, activation=tanh）
  ↓
LSTM Layer 2（N2=32, return_sequences=False）
  ↓
Dense（輸出 2 維：HP 房價指數 + VOL 成交量指數）
```

### 25 個輸入變數（表一）

| 類別 | 變數（6個） | 來源 |
|------|-----------|------|
| 總體經濟 | CSI, GDP, PDI, RHPI, TAIEX, UR | 台灣經濟新報 TEJ |
| 貨幣政策 | M1B, CPI, HIR | 台灣經濟新報 TEJ |
| 房地產特有 | BR, CBL, CCI, HS, NTO, SR | 國泰/內政部/TEJ |
| 人口結構 | DR, LF, NI, POP, PPH | 中華民國統計資訊網 |
| Google Trends | ARR, CL, RE, REA, SVI | pytrends（免費）|

### 資料正規化

```
X_norm = (x - μ) / (x_max - x_min) ∈ [-1, 1]   ← Mean Normalization
```

### 訓練設定

| 項目 | 值 |
|------|-----|
| 研究期間 | 2005Q1 ~ 2020Q3（63季） |
| 訓練資料 | 2005Q1 ~ 2015Q4（44季） |
| 驗證資料 | 2016Q1 ~ 2018Q3（11季） |
| 測試資料 | 2018Q4 ~ 2020Q3（8季） |
| 滾動視窗 | 每年（4季）調整一次模型參數 |
| 最佳超參數 | N1=64, N2=32, Epoch≈200, Batch=16 |

### 論文實證結果（表三 RMSE）

| 模型 | 台灣 HP | 台灣 VOL |
|------|---------|---------|
| GARCH | 1.757 | 519.515 |
| LSTM | 0.118 | 0.049 |
| LSTM 第1次滾動 | 0.008 | 0.019 |
| LSTM 第2次滾動 | **0.004** | **0.012** |

**結論：加入非經濟變數（人口結構 + Google Trends）後，HP 準確度提升 22%，VOL 提升 6%；滾動視窗調整後，準確度分別達 97% 與 76%。**

## 三、論文套用至本專案的調整方案

### 差距分析

| 項目 | 論文做法 | 專案現況 | 需調整 |
|------|---------|---------|--------|
| Layer 2 模型 | 真實 LSTM（25 變數 × 4 落後期） | 線性公式 + sin 波動 | ★★★ |
| 輸出變數 | HP + VOL 雙指數 | 只有 lstm_index 單一值 | ★★★ |
| 滾動視窗 | 每年（4季）重新調參 | Stub 未實作 | ★★ |
| 資料正規化 | Mean Normalization | 未做 | ★★ |
| Layer 3 情緒 | 無（論文不含）| demo_rf_sde.py 固定公式 | 可補強 |

### Layer 2 替換方式

```python
# 替換 demo_lstm.py 的 [REPLACE_LSTM_START/END]

# 現在（Demo）：
lstm_index = 100 * (1 + annual_growth)**11 * (1 + sin波動)

# 替換後（論文版）：
model = load_model("models/lstm_housing.h5")
hp_index, vol_index = model.predict(X_sequence)  # shape=(1,4,25)
lstm_index = hp_index * SCALE_FACTOR
```

### Layer 3 補強方式（利用 VOL 指數）

```python
# demo_rf_sde.py 情緒分數可加入 vol_signal
# VOL 高 → 市場熱絡 → 情緒 +0.05
# VOL 低 → 市場冷淡 → 情緒 -0.05
sentiment_score += vol_signal  # 從 LSTM 輸出取得
```

## 六、王文楷（2025）論文分析

### 論文資訊

- **標題**：利用機器學習和隨機過程預測房價：以台灣房市為例
- **作者**：王文楷（國立高雄大學財務金融學系）
- **出版**：住宅學報，Vol. 34, No. 1，2025年6月

### 與蔡繡容（2023）的核心差異

| 項目 | 蔡繡容（2023） | 王文楷（2025） |
|------|-------------|-------------|
| **輸入變數** | 25 個（需 TEJ/多個 API） | **只有 2 個**（房價指數 + 利率） |
| **資料頻率** | 季資料 | 月資料 |
| **資料來源** | TEJ + 內政部 + 主計總處 + pytrends | **清華安富指數 + 央行**（免費） |
| **模型** | LSTM | LSTM + **RF+SDE** |
| **SDE 用途** | 無 | 模型化長短期市場情緒 |

### RF+SDE 架構（王文楷）

```
GBM（SDE）→ 模型化房價長短期趨勢（n1=30月、n2=10月）
CIR（SDE）→ 模型化利率
  ↓
Monte Carlo → y_k,j = ln(x_long / x_short)（情緒指標，min/mean/max）
  ↓
RF 分類 → 預測未來3個月房價平均成長率方向
```

### RMSE 比較（表三，原始單位）

| 模型 | RMSE |
|------|------|

| LSTM^TS（蔡繡容） | 5.2746 |
| LSTM^TS 第1次滾動 | 0.3576 |
| LSTM^TS 第2次滾動 | 0.1788 |
| **LSTM^W（王文楷）** | 0.7717 |
| **RF+SDE（王文楷）** | **0.3449** |

## 七、鑑價目標釐清與方向轉換

### 重要決策：黑客松 Demo 目標確認

**目標：輸入地址 + 物件類型 → 個別物件估價**

這是**特徵定價法（Hedonic Pricing）**問題，而非市場趨勢預測。

### 現有四層架構 vs. 黑客松需求

| 現有架構 | 適用場景 | 黑客松需要嗎？ |
|---------|---------|-------------|
| Layer 1：靜態縣市單價表 | 個別估價基準 | ✅ 需要（但應升級） |
| Layer 2：LSTM 市場趨勢 | 市場指數預測 | ❌ 不需要 |
| Layer 3：RF+SDE 情緒 | 市場情緒分類 | ❌ 不需要 |
| Layer 4：Monte Carlo | 風險區間模擬 | ✅ 保留做信心區間 |

### 建議方案：XGBoost + 實價登錄

```
使用者輸入：地址（→ 解析行政區）+ 物件類型
  ↓
特徵向量：行政區、建物型態、坪數、屋齡、樓層、停車位
  ↓
XGBoost 回歸（訓練資料：內政部實價登錄公開 API）
  ↓
估價結果（P50）+ Monte Carlo 信心區間（P5/P95）
```

### 文獻支持（參考 docs/taiwan-housing-price-ml-papers.md）

| 論文 | 方法 | MAPE | 資料 |
|------|------|------|------|
| 匿名（2023，台北） | XGBoost + Stacking | **10.17%** | 台北實價登錄 |
