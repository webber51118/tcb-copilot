## XGBoost 鑑價模型說明

### 訓練資料
- **來源**：內政部實價登錄（LVPR）清洗後資料
- **路徑**：`data/lvpr/cleaned_lvpr.parquet`
- **規模**：雙北 ~147,000 筆交易資料

### 特徵工程（10 個輸入特徵）

| 特徵 | 類型 | 說明 |
|------|------|------|
| `district` | 類別（Label Encoded） | 行政區（信義區、大安區…） |
| `building_type` | 類別（Label Encoded） | 建物類型（大樓/公寓/透天） |
| `area_ping` | 數值 | 坪數 |
| `property_age` | 數值 | 屋齡 |
| `floor` | 數值 | 樓層 |
| `total_floors` | 數值 | 總樓層 |
| `has_parking` | 數值（0/1） | 有無停車位 |
| `rooms` | 數值 | 房間數 |
| `year` | 數值 | 交易年份 |
| `quarter` | 數值 | 季別 |

- **目標變數**：`log(price_per_ping)` → 推論後 `np.expm1()` 還原為「元/坪」
- **Log 轉換原因**：房價右偏分佈，log 轉換後接近常態，可降低 MAPE 30~40%

### 超參數設定

```python
XGB_PARAMS = {
    "objective":             "reg:squarederror",
    "n_estimators":          500,
    "learning_rate":         0.05,
    "max_depth":             6,
    "min_child_weight":      5,
    "subsample":             0.8,
    "colsample_bytree":      0.8,
    "reg_alpha":             0.1,
    "reg_lambda":            1.0,
    "early_stopping_rounds": 30,
}
```

### 訓練方式
- **時序切分**（非隨機）：最後 1 年作為測試集，避免未來資料洩漏
- 訓練集 90% 再切 10% 為驗證集（Early Stopping 用）
- 評估指標：MAPE（%）+ 命中率（10% / 20% 誤差內）

## XGBoost 演算法原理

### 核心概念：Gradient Boosting

XGBoost（eXtreme Gradient Boosting）是**集成學習（Ensemble Learning）**方法，屬於 **Boosting** 家族。

```
最終預測 = 樹1預測 + 樹2預測 + 樹3預測 + ... + 樹N預測
```

每一棵樹專門修正前面所有樹的**殘差（誤差）**，逐步逼近真實值。

---

### 與其他模型的差異

| | 決策樹 | Random Forest | XGBoost |
|---|---|---|---|
| 訓練方式 | 單棵 | 平行（Bagging） | 序列（Boosting） |
| 每棵樹學什麼 | 原始資料 | 原始資料 | 前一輪的**殘差** |
| 防止過擬合 | 修剪 | 多樹投票 | 正則化 + Early Stop |

---

### 訓練流程

```
第 1 輪：初始預測 → 計算所有交易的預測誤差（殘差）
         殘差 = log(實際單價) - log(預測單價)

第 2 輪：訓練一棵新樹，專門擬合第 1 輪的殘差
         預測 = 第1棵樹輸出 + learning_rate × 第2棵樹輸出

第 3 輪：訓練一棵新樹，擬合第 2 輪累積後的殘差
         ...

重複 500 輪（n_estimators=500）
每 30 輪若 val loss 不再下降 → Early Stop
```

---

### 目標函數（損失函數）

使用 `reg:squarederror`（均方誤差回歸）：

```
L = Σ(yi - ŷi)²  +  Ω(樹複雜度)
    ─────────────    ──────────────
    預測誤差項        正則化懲罰項
```

**正則化懲罰項**：
- `reg_alpha=0.1` → L1 稀疏化（剔除不重要特徵）
- `reg_lambda=1.0` → L2 平滑化（防止權重過大）

---

### 關鍵超參數解釋

| 參數 | 本專案值 | 作用 |
|------|---------|------|
| `n_estimators` | 500 | 最多長 500 棵樹 |
| `learning_rate` | 0.05 | 每棵樹貢獻縮小至 5%，避免跳過最佳解（Shrinkage） |
| `max_depth` | 6 | 每棵樹最深 6 層（最多 64 個葉節點） |
| `min_child_weight` | 5 | 葉節點最少 5 筆樣本，防止過度細分 |
| `subsample` | 0.8 | 每棵樹隨機抽 80% 資料（Row Sampling，防過擬合） |
| `colsample_bytree` | 0.8 | 每棵樹隨機抽 80% 特徵（Column Sampling） |
| `early_stopping_rounds` | 30 | 連續 30 輪 val loss 無改善則提前停止 |

---

### 本專案模型架構（三層估價）

```
XGBoost（主力）
  └─ 擅長：結構化表格資料、行政區 × 建物特徵交互作用
  └─ 輸入：10 個特徵 → 輸出：log(price_per_ping) → 還原為元/坪

Monte Carlo GBM（Layer 4 信心區間）
  └─ 擅長：模擬價格在未來的隨機漲跌路徑（1,000 次模擬）
  └─ 輸入：XGBoost P50 → 輸出：P5 / P95 信心區間

Demo Fallback（Python 未啟動時）
  └─ 行政區查表 × 建物類型乘數 × 屋齡折舊
  └─ 無需模型，即時回傳估值
```
