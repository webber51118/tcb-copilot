## 二、Bug 修正：Teams 風險因子編號

### 問題

Teams Adaptive Card 的 `TextBlock` 中使用 `${i+1}.` 格式，Teams 渲染時將每個 TextBlock 視為獨立清單，全部顯示為 `1.`。

### 修正

**檔案**：`src/main/typescript/services/powerAutomateNotifier.ts`

```typescript
// 修正前
text: `${i + 1}. ${f.label}（${(f.contribution * 100).toFixed(1)}%）`

// 修正後
const ORDINALS = ['①', '②', '③'];
text: `${ORDINALS[i]} ${f.label}（${(f.contribution * 100).toFixed(1)}%）`
```

## 三、ngrok 設定與限制記錄

### 設定（`C:\Users\Webber\AppData\Local\ngrok\ngrok.yml`）

```yaml
version: "3"
agent:
    authtoken: <token>
tunnels:
  copilot:
    proto: http
    addr: 3000
    domain: idlest-decamerous-audrey.ngrok-free.dev
    request_header:
      add:
        - "ngrok-skip-browser-warning: true"
```

### 限制說明

- `request_header.add` 是將 header 加到「轉發給後端的請求」，無法跳過 ngrok 自身的瀏覽器警告頁
- 正確繞過方式：瀏覽器端發送的 request 帶有 `ngrok-skip-browser-warning` header
- 實際 workaround：使用者在瀏覽器點一次「Visit Site」後，LINE LIFF 頁面即可正常使用
- Express response header middleware（`index.ts`）同樣無效，但無害
