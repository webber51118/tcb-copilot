### S8｜CREW 2：鑑估 PILOT

**標題**：CREW 2 鑑估 PILOT：XGBoost 點估計 × Monte Carlo 壓力情境，15 秒取代 2-3 天

**技術流程**（SmartArt 流程圖）：

```
客戶輸入（地址/坪數/屋齡/格局/樓層）
          ↓
XGBoost 非線性估價
→ 基準點估（如 60 萬/坪）
          ↓
Monte Carlo GBM（1,000 路徑模擬）
→ 壓力情境下的市場波動分佈
          ↓
輸出：P5 / P50 / P95 三情境估價區間
```

**銀行決策邏輯**（SmartArt 三色分層）：
- P5（最壞情境）→ **核貸上限依據**（審慎原則）
- P50（中位數）→ 市場合理估值
- P95（樂觀情境）→ 參考上限

**SHAP 可解釋性輸出**：
- 各特徵貢獻（屋齡 / 樓層 / 捷運距離 / 公設比）
- 對應合庫 AI 治理要求：決策依據可查、可解釋、可重現

**效益數字**：鑑估時間 2-3 天 → **15 秒**（-99.9%）

### S10｜CREW 3：防詐 PILOT

**標題**：CREW 3 防詐 PILOT：8 項規則查核 × ML 異常評分 × SHAP 風險因子

**雙層防詐機制**（SmartArt 左右兩欄）：

**左欄 — 規則引擎（`fraudDetector.ts`）**：
8 項查核指標：
1. 證件比對
2. 公司居住地一致性
3. 職業穩定性
4. 所得成長性
5. 聯徵查詢次數
6. 銀行借款情形
7. 不動產核實
8. 資產淨值合理性

**右欄 — ML 異常評分（`fraudScoringService.py`）**：
- 模型：XGBoost / Isolation Forest
- 輸出：`fraud_score`（0-1）
- 可解釋性：SHAP 前三大風險因子

**三級警示路由**（SmartArt 三色分層）：
- `fraud_score ≤ 0.4` → Level 1：行員一鍵確認
- `fraud_score 0.4-0.7` → Level 2：指派資深行員審查
- `fraud_score > 0.7` → Level 3：🔴 **Teams 主管警示（自動推播）**
