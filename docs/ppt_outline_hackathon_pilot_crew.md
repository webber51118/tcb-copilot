### S6｜PILOT CREW 1：行銷客服

**標題**：PILOT CREW 1 行銷客服：從推薦、交叉銷售到 AI 諮詢，全程陪伴客戶

**左側 — 推薦引擎三層架構**（SmartArt 由下往上）：

**Layer 1 資格篩選（Content-Based Filtering）**：
- 輸入：年齡 / 職業 / 收入 / 貸款用途
- 機制：`isEligible()` 自動過濾，冷啟動問題天然不存在
- 優先序：房貸 青安 → 國軍 → 一般；個貸 國軍 → 薪轉 → 一般

**Layer 2 個人化說明（LLM Instruction Following）**：
- 同一產品，護理師看到「月省 X 元」，軍人看到「軍人專屬優惠」
- 學術依據：InstructRec（arXiv:2305.07001）

**Layer 3 交叉銷售升級 — 合家保加權演算法**：

| 客戶屬性 | 合利超旺（台幣終身）| 美好安心（美元躉繳）| 真美傳家（美元終身）|
|---------|--------------|------------------|---------------|
| 年齡 ≤ 45 | +3 | 0 | +2 |
| 年齡 46-65 | +1 | +3 | +2 |
| 軍公教職業 | +3 | +1 | +1 |
| 月收入 < 6 萬 | +3 | 0 | 0 |
| 月收入 > 10 萬 | +1 | +3 | +3 |
| 青安/軍公教貸款 | +3 | +1 | +1 |
| 以房養老貸款 | 0 | +4 | +2 |

範例：35 歲公務員月薪 8 萬 + 青安貸款 → **合利超旺 11 分**（最高）

**Layer 4 AI 諮詢（RAG + Memory Augmentation）**：
- 觸發：推薦後客戶仍有問題 → 💬 AI 諮詢接棒陪伴
- 機制：知識庫 8 章 + chatHistory 最近 5 輪 → 組 system prompt → LLM 回答
- 效果：Bot 記得客戶年齡、職業、申貸金額，不用重填
- 學術對應：RAG + Memory Augmentation（狀態寫在 context 外面）

**底部 Callout**（紅框）：
> CREW 1 覆蓋完整客戶旅程：**推薦產品 → 合家保交叉銷售 → 台語語音入口 → AI 諮詢跟進**
> 從初次接觸到申辦完成，全程不離 LINE

> | ② 引進軟體顧問廠商整合 LLM 部署 | 浤誠科技建數位中台（微服務化）+ 賽微科技接語音 AI，分工清晰 |

**下半部 — 合庫面臨的三大關卡**（展望，非承諾）：

| 關卡 | 說明 |

> FinLLM 2026 年底上線後，可直接替換底層模型，架構完全相容。

### S19｜創新亮點與差異化

**標題**：四大創新，跳脫傳統銀行 AI 做法

**亮點一：台語語音申辦（全台首創）**：
- Breeze-ASR-26 台語辨識 → LINE Bot 全程中文 UI → 一通語音完成申貸
- 服務對象：400 萬高齡台語族群（傳統數位服務的盲點）
- 差異化：其他銀行 AI 聚焦普通話，台語服務空白

**亮點二：貸款專區一站式入口（優化 UX × 擴大客群）**：
- 整合房貸、信貸、AI 鑑價、優惠專案為單一貸款專區入口
- 客戶不需預判借款類型，開口即被引導到適合方案
- 差異化：傳統銀行 App 分產品各自入口，客戶需先知道要借什麼；貸款專區降低探索成本

**亮點三：合家保加權演算法（跨產品交叉銷售）**：
- 依客戶屬性（年齡/職業/收入/貸款類型）自動選出最適合的理財商品
- 傳統做法：行員靠經驗推薦，缺乏個人化邏輯
- 差異化：貸款 + 壽險 + 理財三產品同步推薦，一次服務最大化

**亮點四：AI 諮詢記憶客服（RAG + Memory Augmentation）**：
- 主選單 💬 AI 諮詢一鍵進入，自由問答不需引導
- Bot 帶知識庫 + chatHistory 上下文，客戶不用每次重填資料
- 符合「AI 客服正確架構」六步驟完整閉環
- 差異化：傳統 FAQ Bot 沒記憶，客戶每次重新開始；我們有 Memory Augmentation

**可延伸至其他部門**：
> 企業金融（中小企業授信）/ 財富管理（投資組合推薦）/ 法金（企業貸款評估）
> 架構可重用，只需替換知識庫與產品目錄
