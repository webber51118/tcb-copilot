### S5｜CREW 1：行銷 PILOT（推薦系統）

**標題**：CREW 1 行銷 PILOT：InstructRec 三層推薦 × 台語語音全台首創

**技術架構**（SmartArt 三層堆疊，由下往上）：

**Layer 1 資格篩選（Content-Based Filtering）**：
- 輸入：年齡 / 職業 / 收入 / 貸款用途
- 機制：`isEligible()` 自動過濾不符資格產品
- 優勢：填表即得完整 User Profile，冷啟動問題天然不存在

**Layer 2 產品排序（Instruction Following）**：
- 房貸優先序：青安 → 國軍 → 一般
- 個貸優先序：國軍 → 薪轉 → 一般
- 主推 70% 版面 + 交叉銷售 30% 版面

**Layer 3 個人化說明（LLM Instruction Following）**：
- AI 大語言模型 即時生成客製化推薦理由
- 同一產品，護理師看到「月省 X 元」，軍人看到「軍人專屬優惠」
- 學術依據：InstructRec（arXiv:2305.07001）

**差異化亮點**（紅框強調）：
> 🎤 台語語音入口：Breeze-ASR-26
> 「我想借錢買厝」→ 直接辨識 → 全程 LINE 內申辦
> **全台首創銀行業台語語音申辦流程**（服務 400 萬高齡族群）

### S6｜CREW 2：鑑估 PILOT（XGBoost 估價）

**標題**：CREW 2 鑑估 PILOT：XGBoost 點估計 × Monte Carlo 壓力情境，15 秒取代 2-3 天

**技術流程**（SmartArt 流程圖）：

```
客戶輸入（地址/坪數/屋齡/格局/樓層）
          ↓
XGBoost 非線性估價
→ 基準點估（如 60 萬/坪）
          ↓
Monte Carlo GBM（1,000 路徑模擬）
→ 壓力情境下的市場波動分佈
          ↓
輸出：P5 / P50 / P95 三情境估價區間
```

**銀行決策邏輯**（SmartArt 三色分層）：
- P5（最壞情境）→ **核貸上限依據**（審慎原則）
- P50（中位數）→ 市場合理估值
- P95（樂觀情境）→ 參考上限

**SHAP 可解釋性輸出**：
- 各特徵貢獻（屋齡 / 樓層 / 捷運距離 / 公設比）
- 對應合庫 AI 治理要求：決策依據可查、可解釋、可重現

**效益數字**：
- 鑑估時間：2-3 天 → **15 秒**
- 估價區間信心水準：P5/P50/P95 三情境

### S7｜CREW 3：防詐 PILOT（規則 + ML 評分）

**標題**：CREW 3 防詐 PILOT：8 項規則查核 × ML 異常評分 × SHAP 風險因子

**雙層防詐機制**（SmartArt 左右兩欄）：

**左欄 — 規則引擎（`fraudDetector.ts`）**：
8 項查核指標：
1. 證件比對
2. 公司居住地一致性
3. 職業穩定性
4. 所得成長性
5. 聯徵查詢次數
6. 銀行借款情形
7. 不動產核實
8. 資產淨值合理性

**右欄 — ML 異常評分（`fraudScoringService.py`）**：
- 模型：XGBoost / Isolation Forest
- 輸出：`fraud_score`（0-1）
- 可解釋性：SHAP 前三大風險因子

**三級警示路由**（SmartArt 三色分層）：
- `fraud_score ≤ 0.4` → Level 1：行員一鍵確認
- `fraud_score 0.4-0.7` → Level 2：指派資深行員審查
- `fraud_score > 0.7` → Level 3：🔴 **Power Automate → Teams 主管警示**

**對應行內邏輯**：
應審慎辦理 / 應特別注意查證 / 強制人工介入（三級防詐分層）

### S8｜M365 整合：Power BI + Power Automate

**標題**：M365 整合：Power BI 行員即時輔助 × Power Automate 兩條自動化流程

**上半部 — Power BI（行員個案即時輔助）**：

受眾：行員（非管理層報表）
用途：三 Crew 推論完成後，即時顯示該案件結果

| Tab | 內容 | 技術 |
|-----|------|------|
| Tab 1 推薦 | CREW 1 產品排序 + 月付金試算 | Power BI REST API `Datasets PostRows` |
| Tab 2 鑑估 | CREW 2 P5/P50/P95 估價區間 + 核貸上限 | Python 推論完成後 push |
| Tab 3 防詐 | CREW 3 警示等級 + SHAP 風險因子 | 即時更新，行員審閱 |

**下半部 — Power Automate（三條自動化流程）**：

```
流程 1：推薦通知 + 行員話術（新增）
PILOT CREW 審核完成
    → Power Automate HTTP trigger
    → Teams 推播行員：推薦方案摘要 + 量身定制話術
      ┌─ 💼 新案件推薦通知｜個金 Co-Pilot
      ├─ 📋 案件 / 客戶 / 金額
      ├─ ✅ 推薦方案：青安貸款　利率：1.565%　月付：22,000 元
      ├─ 📣【行員話術】「王先生，您符合政府青安方案，
      │              利率最優惠，每月只要 22,000 元...」
      └─ ✨ 方案亮點 × 3 + 交叉銷售建議

流程 2：防詐高風險警示
fraud_score > 0.7
    → Power Automate HTTP trigger
    → Teams 推播主管（含案件摘要 + 前三大風險因子）

流程 3：PDF 批覆書自動化
推薦 + 鑑估 + 防詐完成
    → Power Automate 呼叫 /api/generate-pdf
    → 自動存至 SharePoint（案件資料夾）
    → 行員確認簽核即可
```

**價值主張**：
> 從「手動填報 + 自行查找話術」到「AI 推送即時資訊 + 話術直接到位」
> 行員零準備，拿起手機就能開口推薦
