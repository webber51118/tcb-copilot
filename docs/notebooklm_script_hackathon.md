### 第二段｜問題：個金業務三大痛點（0:50–1:40）

畫面：個金業務三代演化圖

傳統個金業務，有三道難關。

第一，服務量能有限。行員一天能服務的客戶有上限，申辦流程繁瑣，等待時間長。

第二，通路受地點限制。第二代線上借貸雖然24小時可申請，但缺乏個人化服務，客戶常常不知道自己符合哪個方案。

第三，主動性不足。業務推廣靠行員主動出擊，難以規模化精準觸及對的客戶。

三代演進讓流程越來越快，但客戶想到貸款就想到合庫的主動性，仍待突破。

我們需要的，是一位永遠在線、主動服務、懂得因人制宜的智慧夥伴。

### 第三段｜解法：Co-Pilot 架構總覽（1:40–2:20）

畫面：Multi-Agent 架構圖，三大 Pilot Crew

個金 Co-Pilot 領航員，建立在 Microsoft Multi-Agent Framework 之上。

核心是一位領航員，下設三個 Pilot Crew。

第一，行銷領航員：演算法精準配對客戶資格，InstructRec 生成個人化推薦話術，行員零準備開口就推薦。

第二，鑑估領航員：XGBoost 機器學習，15秒給出三情境估價，全程可解釋，行員直接引用核貸。

第三，防詐領航員：八項規則快篩加上AI深度評分，兩層把關，fraud score 即時推送主管手機。

整套系統透過 LINE Bot 直接觸達客戶，無需下載App、無需臨櫃。
行員掌舵，Agent 全速執行。

### 第四段｜CREW 1 Demo：行銷推薦 + LINE Bot 完整流程（2:20–3:20）

畫面：LINE 聊天室截圖，以房養老試算卡片

讓我們看一個真實的使用情境。

一位客戶在 LINE 開啟合庫個金 Co-Pilot，詢問以房養老方案。

系統即時回傳試算結果卡片。分段式利率2.338%，一段式2.608%，預估月付金新台幣1,934元。卡片下方清楚標示申辦資格，一鍵預約最近分行。

同時，Co-Pilot 根據客戶年齡與貸款需求，自動推薦房貸壽險與合利超旺理財商品。這是行銷領航員的交叉銷售演算法在背後運作，35歲公務員加月薪八萬加青安貸款，系統算出合利超旺11分，第一推薦。

畫面：線上申請書步驟進度條，電子簽名畫面

最關鍵的環節，線上申請書。

客戶點選填寫申請書，系統帶出四步驟電子表單：確認貸款資訊、補充個人資料、同意條款、手寫電子簽名。

從點擊到完成簽名，全程在 LINE 內完成，不需切換App。

案件編號 TCB-20260517-0001，合庫三到五個工作天內聯繫，流程清晰，體驗完整。

### 第五段｜CREW 2 Demo：AI 鑑估，15秒三情境估價（3:20–4:10）

畫面：AI 鑑估輸入頁面，鑑估結果卡片

傳統人工鑑估，要2到3天。更嚴重的是，同一物件，A行員估900萬，B行員估1,050萬，估值主觀，缺乏可稽核依據。稽核來問依據是什麼，行員只能說，這是我的判斷。

鑑估領航員的答案是：機器學習，客觀量化。

系統以近12季、雙北41個行政區、逾60萬筆真實成交資料為基礎，訓練 XGBoost 梯度提升模型。500棵決策樹接力學習，每棵專門修正上一棵的誤差，誤差越來越小。

客戶輸入物件資訊，坪數、屋齡、樓層、建物型態。

畫面：三情境估價結果 P5 / P50 / P95

15秒後，三情境估價出爐。

P5 最保守，1,454萬，這是銀行核貸上限的依據，審慎原則保護銀行債權。
P50 市場中值，1,659萬，這是客戶期望的參考基準。
P95 最樂觀，1,897萬，供行員判斷裁量空間。

畫面：SHAP 因子解釋

AI 同步說明估價依據。SHAP 拆解前三大因子：總樓層數影響估價負24%，樓層加18%，成交年份加20%。

每一個數字都有依據，每一筆決策都可被稽核，符合合庫 AI 治理要求，決策依據可查、可解釋、可重現。

鑑估時間，從2到3天縮短為10分鐘。

### 第六段｜MCP 串接：Twinkle Hub 政府開放資料（4:10–4:50）

畫面：Copilot Studio MCP 工具清單，實價登錄查詢 Demo

機器學習的估價是基於訓練資料。但如果行員今天想查最新的實價登錄行情，怎麼辦？

這就是我們在今年5月12日完成的新突破，MCP 串接。

MCP，Model Context Protocol，是 AI Agent 的資料接頭。任何 AI Agent 都能直接接上，設定一次，永久可查。

我們透過 Twinkle Hub MCP Server，將台灣52,960筆政府開放資料整合為一個端點，涵蓋不動產、財務、法律、交通等20個領域。

個金 PILOT 行員助理今天就能直接問：查台北市中和區近期實價登錄行情。

畫面：Agent 呼叫 MCP，回傳各樓層成交均價

Agent 自動呼叫 Twinkle Hub MCP，回傳各樓層真實成交均價。

不是AI憑記憶猜測，而是從52,960筆政府資料即時回答。不幻覺，不靠記憶。

資料來源 OGDL-Taiwan-1.0 開放授權，合法合規。這不是未來展望，今天就能Demo。

### 第七段｜亮點：台語語音辨識（4:50–5:30）

畫面：台語語音錄音介面，辨識結果卡片

我們特別為高齡客群設計了台語語音輸入。

合庫的傳統存款客群，台語是主要的日常溝通語言。但所有銀行的App與LINE Bot介面，均以普通話為主，台語族群面臨數位障礙。對手銀行的AI全面聚焦普通話，台語服務空白一覽無遺。

一位70歲的長輩，用台語說：啊這間厝喔，差不多六十坪，想要貸款借錢，敢無法度？

系統透過聯發科 MediaTek Research 開發的 Breeze-ASR-26 台語辨識模型，以10,000小時台語語料訓練，Apache 2.0 開源，零授權費用，搭配 Claude NLU 語意理解，在三到五秒內將台語即時轉換為標準中文，並自動判定貸款類型為房屋貸款，引導後續填寫流程。

合庫用開源模型，零授權費，率先服務台語族群。全台銀行業首創。

語言，不再是數位金融的門檻。

### 第八段｜CREW 3 防詐 + Power BI 儀表板（5:30–6:00）

畫面：Teams 主管即時推播通知，Power BI 儀表板四Tab

防詐領航員提供兩層把關。

第一層，八項規則引擎。身份文件比對、聯徵查詢次數、所得成長性、資產淨值合理性等八項指標，進件即觸發，有就是有，不靠行員記憶，不落灰色地帶。

第二層，AI 深度評分。XGBoost 輸出0到1的 fraud score，SHAP 拆解前三大風險因子，主管手機秒收 Teams 警示，不需開系統就能即時介入，滿足強制人工介入的合規要求。

所有案件即時彙整於 Power BI 四Tab 儀表板。

其中最反直覺的洞察，AI Key Influencers 自動發現：貸款金額下降427萬時，高風險警示機率反而上升2.48倍。小額信貸的詐欺評分更集中在高風險，這是人工審核容易忽略的盲點。

傳統邏輯，金額小等於風險低，在這裡被AI推翻了。
