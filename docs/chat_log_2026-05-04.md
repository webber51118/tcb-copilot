## 二、中文亂碼診斷與修正

### 問題根因

第一次用 curl（Git Bash）推送中文資料時，Windows 終端機編碼非 UTF-8，導致：
- `青安貸款` → `◆@◆◆жU`
- `房屋貸款` → `◆bIZU◆◆`

報表標題（手動輸入）顯示正常，但資料欄位值（從資料集讀取）全部亂碼。

### 解決方案

建立 `scripts/push-test-data.mjs`，用 Node.js 原生 UTF-8 確保中文正確推送：

```bash
POWER_BI_PUSH_URL="..." node scripts/push-test-data.mjs
# HTTP 200 ✅ 成功推送 6 筆資料
```

### 資料集重建

串流資料集無法刪除舊資料，且舊亂碼資料與新正確資料混在一起（卡片顯示 12 筆而非 6 筆）。

**解決**：刪除舊資料集，重新建立，同時修正 Schema：

| 欄位 | 舊型別 | 新型別 | 原因 |
|------|--------|--------|------|
| `AlertLevel` | Int64 | **文字** | 實際值為 "LOW"/"MEDIUM"/"HIGH" |

新 Dataset ID：`9c70aaa4-9215-415c-bb5b-817e483b2f13`

## 六、Power BI ML 功能可行性討論

### 6-1 評估背景

評估三大 Power BI ML 方式是否可用於黑客松：AutoML、Python/R 腳本、AML 整合，以及內建 AI 視覺效果。

### 6-2 核心限制

專案使用 **Push 串流資料集**，此類型不支援 Power BI 進階 AI 視覺效果：

| AI 視覺效果 | 適合用途 | 支援 Push 串流？ |
|------------|---------|----------------|
| 異常偵測 | 審核量時間趨勢尖峰偵測 | ❌ 不支援 |
| 關鍵影響因素 | 分析驅動 FraudScore 的因素 | ❌ 不支援 |
| 分解樹 | 切分貸款類型 → 風險等級 | ❌ 不支援 |

### 6-3 三大 ML 方式判斷

| 方式 | 結論 | 原因 |
|------|------|------|
| AutoML（Dataflows）| ❌ 不可行 | 需 Power BI Premium 授權 |
| Python/R 腳本（Power Query）| ❌ 無附加價值 | 需 Desktop + Import 資料集，ML 已在 FastAPI 完成 |
| Azure ML 整合 | ❌ 時間成本過高 | 模型未部署至 AML，設定需數天 |

### 6-4 決策：選方案 A — 建立平行 Import 資料集

**做法**：保留 Push 資料集供即時磁磚，另建 Import 資料集展示 AI 視覺效果。

**實作步驟**：
1. 將 6 筆測試資料（`scripts/push-test-data.mjs`）匯出為 CSV
2. Power BI Service → 建立新 Import 資料集（上傳 CSV）
3. 新增第四個 Tab「AI 洞察」：
   - **Key Influencers**：分析欄位 FraudScore，解釋欄位 LoanType / LoanAmount / InterestRate
   - **Decomposition Tree**：FraudScore → LoanType → AlertLevel
4. 釘選至儀表板「個金 COPILOT 即時監控」

**預估工時**：1-2 小時
