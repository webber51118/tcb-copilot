## 二、powerBiService.ts 重構

### 問題背景

昨日建立 Power BI 串流資料集後，程式碼仍使用 Service Principal OAuth 模式（需 Azure AD 申請）。黑客松期間 Azure AD 權限申請耗時，改採平台直接提供的 Push URL（含 Key）。

### 變更內容

| 項目 | 舊版（Service Principal） | 新版（Push Key） |
|------|--------------------------|-----------------|
| 認證方式 | OAuth2 Client Credentials | Push URL 含 Key |
| 環境變數 | 5 個（TENANT_ID / CLIENT_ID / CLIENT_SECRET / WORKSPACE_ID / DATASET_ID）| 1 個（PUSH_URL）|
| 程式碼行數 | 122 行 | 77 行（-45 行）|
| Azure AD 依賴 | 有 | 無 |
| `getAccessToken()` | 有 Token 快取邏輯 | 移除 |
| POST body 格式 | `{ rows: [row] }` | `[row]`（陣列）|

### Commit

```
refactor(powerbi): 改用 Push Key 模式，移除 Service Principal 依賴
0781982
```

### Tab 2｜鑑價分析

| 視覺化 | 類型 | 欄位 |
|--------|------|------|
| 鑑價 P5/P50/P95 區間 | 群組直條圖 | ApplicationId + 三個 Valuation 欄位 |

| 平均鑑價（P50） | 卡片 | AVG(ValuationP50) |

> 頁面層級篩選器：LoanType = 房屋貸款（信貸案件 ValuationP50=0 排除）

### Tab 3｜防詐警示

| 視覺化 | 類型 | 欄位 |
|--------|------|------|
| 最高風險分數 | 量測計 | MAX(FraudScore)，目標線 70 |
