## 三、技術架構

**檔案位置**：`public/index.html`（Express 靜態服務）
**存取路徑**：`http://localhost:3000/dashboard/`

**技術選型**（無框架，CDN 引入）：
- Tailwind CSS CDN — 樣式
- Chart.js CDN — 水平長條圖（推薦商品分布）
- 原生 fetch — 每 5 秒輪詢後端 API

**使用的後端 API**：
- `GET /api/admin/agents/status` — PILOT CREW 三引擎 + 8 個 Agent 狀態
- `GET /api/admin/applications` — 案件清單（28 筆）

## 四、儀表板功能

### 區塊配置

```
┌──────────────────────────────────────────────────────┐
│  Header：個金 Co-Pilot — PILOT CREW 即時監控中心     │
├─────────┬────────────────────────────────────────────┤
│  側邊欄  │  KPI 卡片 ×4（案件數/詐欺分/審核時間/呼叫）│
│  篩選器  ├────────────────────────────────────────────┤
│  貸款類型 │  PILOT CREW 狀態 | 風險四象限 | 商品分布圖 │
│  案件狀態 ├────────────────────────────────────────────┤
│         │  即時案件 Feed（20 筆，可篩選，每 5 秒更新）│
└─────────┴────────────────────────────────────────────┘
```

### 功能細節

| 區塊 | 內容 |
|------|------|
| 4 個 KPI 卡 | 今日 28 案件、最高詐欺 91 分、AI 審核 2.4 秒、Agent 呼叫 185 次 |
| PILOT CREW 狀態 | 三引擎成功率進度條（對話推薦引擎/XGBoost鑑價/5P徵審）+ 5 個輔助 Agent |

| 即時案件 Feed | 時間/案件編號/申請人/類型/金額/狀態，可依貸款類型和案件狀態篩選 |

## 五、修正過程

### 問題一：圖表高度失控
- **原因**：Chart.js canvas 沒有固定高度容器，隨父容器拉高
- **修正**：外層加 `style="position:relative;height:210px;"` + 改用水平長條圖（`indexAxis:'y'`）

### 問題二：移除 Claude AI 字樣
- Header 副標移除「Claude AI」，保留「合作金庫銀行 × Microsoft 365」

### 問題三：未通過案件拒件原因

**定義說明**：
| 狀態 | 定義 |
|------|------|
| `pending` | 客戶剛送出申請，等待審核 |
| `reviewing` | PILOT CREW 已審，行員複核中 |
| `approved` | 已核准，準備撥款 |
| `rejected` | 未通過（DBR超22倍/負債比>85%/詐欺高分等）|

**實作**：
- `LoanApplication` 新增 `rejectionReason?: string` 欄位
- `data/applications.json` 兩筆 rejected 案件加入實際原因：
  - 古林睿煬（信貸 30 萬）：DBR 超過 22 倍（現欠合計達月收入 24.3 倍）
  - 陳金鋒（房貸 2,000 萬）：負債比超過 85%（房貸本息 + 現有債務 / 月收入 = 91.2%）
- 儀表板 Feed 顯示截短原因文字，hover 顯示完整說明
