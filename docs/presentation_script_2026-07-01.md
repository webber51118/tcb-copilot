## 完整口說講稿

---

### 【投影片 1｜0:00–0:30】封面

分行行員每天在做三件事：

幫客戶找到最適合的貸款產品、估算擔保品的價格、以及防範詐欺。

這三件事，過去靠經驗、靠電話、靠眼力。

今天，我們要用 AI，全部接手。

（換頁）

---

### 【投影片 2｜0:30–1:30】多 Agent 架構總覽

這就是「個金 Co-Pilot 領航員」的架構。

我們設計了三支 Pilot Crew：

Crew 1，**行銷領航員**，負責找到對的客戶、推對的產品；

Crew 2，**鑑估領航員**，用機器學習自動估房價；

Crew 3，**防詐領航員**，即時偵測高風險案件。

三支 Crew 統一透過 LINE Bot 服務客戶、透過 Power BI 讓行員掌握全貌、透過 Copilot Studio 和 Power Automate 串接 M365 工作流程。

這是整個系統的地圖，後面每一個功能，都在這張圖裡。

（換頁）

---

### 【投影片 3｜1:30–1:50】痛點一：手動推薦的困境

傳統個金申貸走了三個世代——紙本、電子表單、到現在——

行員仍靠人工經驗推薦產品。

沒有輔助工具，行員不知道哪個客戶適合哪個商品。

這就是 Crew 1 要解決的問題。

（換頁）

---

### 【投影片 4｜1:50–2:30】LINE Bot 客戶旅程

客戶打開 LINE，看到六宮格選單。

申請房貸、申請信貸、查詢進度、法規問答，全都在 LINE 裡完成。

Co-Pilot 在背後問幾個問題——年齡、收入、用途、貸款金額——

收集完畢，三支 Crew 同步啟動。

（換頁）

---

### 【投影片 5–6｜2:30–3:30】Crew 1 推薦引擎 + 交叉銷售

Crew 1 做的事，是「**用對的話說給對的人聽**」。

它先幫行員篩選：這個客戶符合哪個產品的資格？青安、國軍，還是一般房貸？

確定產品後，系統生成個人化銷售話術——

不是罐頭文字，是根據這個客戶的條件量身撰寫的。

最後，它還會建議跨售商品：房貸壽險保障還款、搭配信用卡現金回饋。

★效益：行員不需要手動判斷，Co-Pilot 一秒給出推薦與話術，讓每一通客戶電話都更有效率。

（換頁）

---

### 【投影片 7–9｜3:30–4:15】M365 整合：Teams + 法規問答 + 壽險跨售

完成推薦之後，系統自動串接 M365。

Power Automate 把案件進度通知推到行員的 Teams，不用另外登入系統。

Copilot Studio 扮演兩個角色：

一是**法規問答機器人**——行員問「這個案子的 LTV 上限是多少」，它立刻查央行規定回答；

二是在對話中**自然帶入房貸壽險跨售**，讓業績機會不因為流程繁瑣而流失。

（換頁）

---

### 【投影片 10｜4:15–4:45】業界首創台語 ASR

這裡有個業界第一。

客戶可以用**台語**講貸款需求。

我們整合了 MediaTek Breeze-ASR-26 台語語音辨識模型——

三秒的語音，11 秒內辨識完成，自動轉成標準中文，並判定貸款類型。

★創意：這是業界首創將台語語音辨識整合進個金服務流程——台語說、中文記，服務台語族群不再是障礙。

（換頁）

---

### 【投影片 11｜4:45–5:00】痛點二：人工鑑估的瓶頸

房屋鑑估是申貸最卡關的環節。

行員打電話、等外部鑑定、再核算，動輒花好幾天。

Crew 2 用 AI，在幾秒內完成。

（換頁）

---

### 【投影片 12–14｜5:00–6:15】Crew 2 鑑估領航員 + MCP + SDE

Crew 2 鑑估領航員的核心，是**資料源**。

我們串接了 **Twinkle Hub 政府開放資料 MCP**，取得實價登錄、地籍、建照等公開資訊，直接在 Copilot Studio 裡呼叫。

估價模型用 XGBoost 訓練雙北 12 季、41 個行政區、10 個特徵的成交資料，

搭配 SDE 蒙地卡羅 **1,000 路徑**模擬，輸出三種情境：

- P5 保守估
- P50 中性估
- P95 樂觀估

★技術：準確率 **93.35%**，誤差 **0.6% 以內**，SHAP 分析告訴行員前三大估價因子——屋齡、樓層、行政區。從輸入地址到拿到鑑估報告，全程自動。

（換頁）

---

### 【投影片 15｜6:15–6:25】痛點三：詐欺偵測靠人工

詐欺核貸是審核流程最危險的盲點。

傳統靠人工複核，速度永遠跟不上詐欺手法的演進。

Crew 3 在申請**當下**就預警。

（換頁）

---

### 【投影片 16–17｜6:25–7:30】Crew 3 防詐領航員 + Power BI 儀表板

Crew 3 防詐領航員設計了兩層防線。

**第一層：8 項規則偵測**

DBR 超限、年齡不符、收入異常、連絡電話可疑——任一觸發立即標記。

**第二層：AI 詐欺評分模型**

輸出 0 到 100 的風險分數，並列出前幾大風險因子。

結果即時送到 Power BI 儀表板：

高風險案件自動標紅，行員一眼看到哪些案子需要優先複核。

★效益：不需要等到審核結束才發現問題，在申請**當下**就預警——高風險案件，零漏網。

（換頁）

---

最後這張，是我們的成績單。

## Q&A 題庫（備用對稿）

| 可能問題 | 建議回答方向 |
|---------|------------|
| **資料安全性？** | Azure Cosmos DB 存儲、LINE LIFF 驗證、符合金融資安規範，客戶資料不落地於外部伺服器 |
| **模型準確率怎麼驗證？** | 使用雙北 12 季實價登錄資料，劃分 Out-of-sample 測試集獨立驗證，93.35% 準確率為測試集結果 |
| **台語 ASR 支援哪些方言？** | Breeze-ASR-26 覆蓋台灣主要腔調（閩南語各腔調），支援中台混語辨識 |
| **導入成本？** | LINE Bot + M365 + Azure 雲端服務，無需硬體投資，採用訂閱制，可隨分行規模彈性調整 |
| **推廣到其他分行？** | 設計即考量擴展性——LINE 官方帳號 + API 架構可複製，只需設定對應行員帳號即可上線 |
| **DBR 計算公式？** | 個人信貸採 DBR ≤ 22 倍，房貸採（每月還款 + 18,000）/ 月收入 ≤ 85%，符合央行規定 |
| **Twinkle Hub MCP 是什麼？** | 政府開放資料的 MCP 介接層，讓 AI Agent 可直接呼叫內政部不動產成交資訊、地籍圖等公開資料庫 |
| **多 Agent 框架選型？** | 採用 Harness Engineering 框架，支援並行 Crew 協作、任務路由與結果彙整，已在生產環境驗證 |
