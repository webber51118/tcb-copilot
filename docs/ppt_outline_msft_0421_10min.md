個金部現行規劃（斜體小字）：

⚙️ 中欄【中端（營運斷點）】
卡片標題：規則引擎撞上自動化天花板

個金部現行規劃（斜體小字）：
自動過件率目標 2.5 成 → 5–7 成
MyData 所得 / 勞保資料自動帶入
**收支表、資產負債表自動化整合（著重方向）**
不動產自動估價系統（與授管部協商中）

現實與根因：
• 規則 IF/THEN 設計保守，稍微複雜的案件就落人工
• MyData 能帶入資料，但系統無法「理解」資料
• 鑑估還是靠行員人工輸入合理單價，結果因人而異
• 根因：規則無法學習，每次法規更新都要人工改參數

Co-Pilot 補足（綠色小字）：
→ XGBoost 非線性學習 + Critic Agent 灰色地帶裁量
→ Monte Carlo 鑑估取代人工查詢，15 秒出結果
→ 收支表 / 資產負債表自動解析，MyData 直接結構化輸入模型

Co-Pilot 補足（綠色小字）：
→ MD + YAML 結構化讓所有資料 AI 可讀（AI-readable）
→ Azure AI Search + Cosmos DB
→ 總行 Power BI 即時看所有分行，貸後自動告警

### 投影片 4【前端解法：LINE Bot + 台語語音 + OCR】
標題：前端解法：台語語音入口 × LINE 完整申貸閉環 × 文件 OCR 免輸入

對應斷點：前端（獲客斷點）— 數位體驗斷裂與高齡落差

現行困境：
點擊 → 彈窗 → 跳 App → 放棄（進件前即流失）
400 萬高齡長輩無法適應 App，台語服務完全缺席
行員手動輸入所得證明 / 謄本資料，每件耗時 20–40 分鐘

Co-Pilot 三項解法：

① LINE 完整閉環
諮詢 → 推薦 → 申辦 → 追蹤 → 核准，全程不離開 LINE（< 3 分鐘）
不是 LINE Bot 導流——是 LINE 內完整閉環，申辦完成不需打開任何 App

② 台語語音入口
「我想借錢買厝」直接辨識，Breeze-ASR-26 本地推論
65 歲以上族群 400 萬人，Financial Inclusion，對齊金管會普惠金融政策

③ 文件 OCR 自動解析
所得證明 / 土地謄本上傳 → Azure AI Document Intelligence 自動 OCR
資料自動填入徵審系統，行員免手動輸入，錯誤率降至接近零

### 投影片 5【中端解法：三層推薦引擎 × 產品全覆蓋】
標題：中端解法：三層推薦引擎 × 新舊客戶智慧分流 × 全產品線覆蓋

對應斷點：中端（營運斷點）— 規則引擎撞上自動化天花板

三層架構：
Layer 1：Content-Based Filtering — 資格過濾（規則層，快速排除不合格產品）
Layer 2：Collaborative Filtering — 生命週期套餐（Netflix 演算法）
  「申辦青安的客戶 68% 同時辦了御璽卡 + 房貸壽險」→ 套餐自動組合
Layer 3：LLM Instruction Following — Qwen2.5 個人化說明
  學術依據：InstructRec（ArXiv 2023）；每個客戶看到不同的推薦說明

新舊客戶分流：
舊客戶 → Collaborative Filtering（有歷史資料，精準套餐推薦）
新客戶 → InstructRec Cold Start（填表即下指令，不需歷史資料，冷啟動天然不卡關）

全產品線覆蓋（組合數量）：
房貸（4種）× 信貸（3種）× 信用卡（4種）× 保險（6種）× 財富管理（3種）× 信託（3種）
→ 潛在組合超過 800 種，傳統行員背不完，Co-Pilot 秒配

生命週期套餐示範：
• 首購族（30歲/公務員）→ 青安房貸 + 御璽卡 + 房貸壽險 + CoBaby親子帳戶
• 退休族（62歲/自有房）→ 以房養老 + 長照保險 + 安養信託 + 黃金存摺
• 職業軍人 → 國軍輔導房貸 + 軍人認同卡 + 軍人團體保險
