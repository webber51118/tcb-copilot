## 四、架構釐清：Pilot Crew 是 Agent 嗎？

### 結論

> **現在的 Crew 是「朝 Agent 方向設計的 Service 模組」，不是真正的 Agent。**

---

### Agent 的真正定義

```
感知環境 → 自主決策（LLM 推理）→ 選擇工具 → 執行 → 觀察結果 → 再決策
```

真正 Agent 的五大特質：

| 特質 | 說明 |
|------|------|
| 自主性 | 自己決定要做什麼、呼叫哪個工具 |
| Tool Use | 動態選擇並呼叫外部工具/API |
| 規劃能力 | 多步驟推理，達成目標 |
| 適應性 | 根據中間結果調整後續行動 |
| LLM 驅動 | 決策來自語言模型，不是寫死的 if/else |

---

### 各 Crew 現況評估

| Crew | 現況 | 本質 |
|------|------|------|
| Crew 1 徵信 | TypeScript 規則引擎，5P 計分寫死 | Rule-based Service |
| Crew 2 鑑估 | 呼叫 Python FastAPI，回傳數字 | API Client |
| Crew 3 法規 | BM25 檢索 + Claude 合成 | **接近 Agent**（有 LLM 推理）|
| Crew 4 審議 | 彙整結果，規則判斷核准/婉拒 | Aggregator Service |
| Crew 5 文件 | Claude Vision 解析圖片 | **接近 Agent**（有 LLM 解讀）|
| Crew 6 客服 | LINE Bot 對話狀態機 + Claude | **接近 Agent**（有 LLM 對話）|
| Crew 7 防詐 | 規則計分 + Mock 資料 | Rule-based Service |

---

### 三層架構演進路線

```
Level 1  Pipeline（現在的狀況）
─────────────────────────────
workflowService 寫死呼叫順序
每個 Crew 接收固定輸入、回傳固定輸出
沒有自主性，沒有 LLM 決策

Level 2  Orchestrator + Workers（黑客松目標）
─────────────────────────────
指揮艙（Orchestrator）用 LLM 決定要召集哪些 Crew
各 Crew 仍是確定性模組
→ 整體行為更靈活，但 Crew 本身還不是 Agent

Level 3  真正的 Multi-Agent（協理規格書長遠規劃）
─────────────────────────────
每個 Crew 都是獨立 Agent
有自己的 LLM + Tool Set + Memory
Agent 間可以相互溝通、協商
→ 這才是 MAF / AutoGen / LangGraph 的真正應用
```

---

### 為什麼黑客松仍適合用「Agent」包裝

1. **架構邊界正確** — 每個 Crew 職責單一、輸入輸出明確，是實現真正 Agent 的最佳基礎
2. **部分 Crew 確實有 LLM 推理** — Crew 3/5/6 已在使用 Claude，有 Agent 的影子
3. **擴充路徑清晰** — 把 Crew 1 的規則引擎換成「LLM + Tool Use」即升級為真 Agent
4. **概念易懂** — 向評審說明「7 個 AI Agent 協作」比「7 個 Service 模組」更有說服力

---

### 升級為真正 Agent 的路徑（以 Crew 1 為例）

**現在（Rule-based）：**
```typescript
function scoreBorrowerProfile(data) {
  if (age < 25) score -= 10;
  if (income > 100000) score += 20;
  return score;
}
```

**升級後（LLM Agent）：**
```typescript
const crew1Agent = new Agent({
  system: "你是徵信專家，依據 5P 原則分析借款人信用",
  tools: [queryJointCredit, checkMyData, calculateDBR],
  // LLM 自行決定要呼叫哪些工具、呼叫幾次
});
const result = await crew1Agent.run(applicationData);
```

升級優先順序：
1. **Crew 7 防詐**（設計即考慮 LLM，模擬資料可直接替換）
2. **Crew 1 徵信**（規則最複雜，Agent 化效益最大）
3. **Crew 4 審議**（彙整推理，適合 LLM 做最終判斷）
