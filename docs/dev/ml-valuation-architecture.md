## 一、系統定位

ML 鑑價引擎為個金 Co-Pilot 的不動產估價核心，負責在房貸徵審流程中提供：

- **估值基準**（基準估值 → LSTM 調整 → RF+SDE 情緒調整）
- **風險區間**（Monte Carlo GBM P5/P50/P95）
- **LTV 計算**（貸款金額 ÷ P50 估值）

鑑價結果直接影響授信成數決策，須符合 FSC 可解釋性要求。

## 二、現有架構（Demo 模式）

```
ValuationRequest（縣市、建物型態、屋齡、坪數、格局、樓層、停車位、貸款金額）
        │
        ▼
┌───────────────────────────────────┐
│  Layer 1：region_price_table      │  縣市單價 × 坪數 × 各係數 → base_value
└───────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────┐
│  Layer 2：demo_lstm.py            │  線性成長率 + sin 季節波動（數學近似）
│  輸入：縣市成長率參數             │  → lstm_adjusted_value, lstm_index
└───────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────┐
│  Layer 3：demo_rf_sde.py          │  斜率公式 + 建物需求修正 + 屋齡修正
│  輸入：縣市、建物型態、屋齡       │  → rf_adjusted_value, sentiment_score
└───────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────┐
│  Layer 4：monte_carlo.py          │  GBM 1,000 路徑模擬
│  μ=2%, σ=8%, T=1yr, n=252 steps   │  → P5 / P50 / P95, risk_level
└───────────────────────────────────┘
        │
        ▼
ValuationResult（estimated_value, confidence_interval, ltv_ratio, risk_level）
```

### Demo 模式限制

| 層次 | 問題 |
|------|------|
| Layer 2 LSTM | 使用固定成長率公式，無法反映真實市場動態 |
| Layer 3 RF | 情緒分數為數學近似，未使用真實訓練資料 |
| 整體 | 無法隨市場變化自動更新，預測能力有限 |

### 3.4 LSTM 模型架構

```
輸入層：shape = (batch, 4, 25)   ← 4 季落後期 × 25 特徵
    │
    ▼
LSTM Layer 1：隱藏神經元 N1（含遺忘閥 Forget Gate）
    │  return_sequences=True
    ▼
LSTM Layer 2：隱藏神經元 N2
    │  return_sequences=False
    ▼
Dense Layer：輸出維度 2
    │
    ▼
輸出：[HP_predicted, VOL_predicted]
```

**超參數（由 Rolling Window 試誤法調整）：**

| 參數 | 初始值 | 調整範圍 |
|------|--------|----------|
| 神經元數 N1 | 64 | 32 ~ 128 |
| 神經元數 N2 | 32 | 16 ~ 64 |
| 訓練次數 epoch | 200 | 100 ~ 500 |
| Batch size | 16 | 8 ~ 32 |
| 激勵函數 | tanh | tanh / ReLU |
| 損失函數 | MSE | — |
| 優化器 | Adam | — |

### 3.5 資料正規化（平均值正規化）

論文採用平均值正規化（Mean Normalization），公式如下：

```
X_norm = (X_t - μ) / (X_max - X_min)

其中：
  X_t   = t 時點之變數值
  μ     = 變數平均值
  X_max = 最大值
  X_min = 最小值
```

正規化後所有變數收斂為 [-1, 1] 的浮點數，避免因量綱差異造成模型偏誤。

### 3.7 滾動視窗機制（Rolling Window）

論文最重要的貢獻之一，可使模型與時俱進：

```
[初始訓練] 2005Q1-2015Q4
      │
      ▼
[第 1 次測試] 2018Q4-2019Q3
      │ 試誤法調整：神經元數、訓練次數
      ▼
[第 1 次 Rolling] 加入 2018Q4-2019Q3 資料重新訓練
      │
      ▼
[第 2 次測試] 2019Q4-2020Q3
      │ 再次調整參數
      ▼
[第 2 次 Rolling] 加入最新一季資料重新訓練
      │
      ▼
[持續每季執行 → 自動化排程]
```

**效果（論文實證）：**

| 指標 | 未滾動 LSTM | 滾動後 LSTM |
|------|-------------|-------------|
| 房價指數 RMSE（台灣） | 0.118 | 0.008 |
| 成交量 RMSE（台灣） | 0.049 | 0.004 |
| 房價指數準確度 | 93% | **97%** |
| 成交量準確度 | 61% | **76%** |

## 四、與 LLaMA 等開源 LLM 的定位分工

| 任務 | 最適模型 | 說明 |
|------|----------|------|
| 市場指數時序預測 | **LSTM（本架構）** | 數值回歸任務，LSTM 原生擅長 |
| 市場情緒分析（結構化特徵）| **Random Forest** | 表格資料，高可解釋性 |
| 市場新聞情緒萃取 | LLaMA（輔助） | 非結構化文字 → 情緒分數 |
| 物件描述特徵解析 | LLaMA（輔助） | 仲介文字 → 結構化 feature |
| 鑑價報告文字生成 | LLaMA / Claude | 數值結果 → 人讀說明 |
| 風險區間模擬 | Monte Carlo GBM | 隨機過程，保留現有架構 |

> **重要**：LLaMA 不適合作為數值預測核心，原因：
> 1. FSC 法規要求授信模型需具備可解釋性，LLM 黑箱難以合規
> 2. LLM 容易產生幻覺數字，鑑價金額錯誤可能造成授信損失
> 3. LLaMA 推論速度為秒級，不符合即時鑑價需求

## 六、評估指標

依論文方法，以 RMSE 為主要衡量指標：

```
RMSE = sqrt( (1/n) × Σ (y_pred - y_actual)² )
```

| 目標 | 房價指數 RMSE | 成交量 RMSE |
|------|---------------|-------------|
| Demo 模式（當前） | 無法衡量（固定公式） | 無法衡量 |
| Phase 2 目標（無滾動） | < 0.12 | < 0.05 |
| Phase 3 目標（有滾動） | < 0.01 | < 0.005 |
| 論文基準（台灣地區）| 0.008 | 0.004 |
