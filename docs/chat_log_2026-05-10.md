## 四、AI 諮詢功能規劃

### 需求背景

黑客松錄影 Demo 需要展示 AI 客服概念（RAG + Memory Augmentation）。
參考 @kai_ch_chen Threads 貼文的「AI 客服正確架構」六步驟設計。

### 現有 Bot 問題診斷（對應圖四）

**B 型問題**：撈了沒寫回 → 下次又忘
- Session in-memory，TTL 30 分鐘，重啟歸零
- 申請案件有寫 JSON，但對話歷史沒有持久化

### 功能設計（錄影 Demo 範疇）

| 項目 | 設計 |
|------|------|
| 觸發方式 | 主選單第 6 格改為「💬 AI 諮詢」，或直接輸入「AI諮詢」|
| 記憶範圍 | session in-memory（錄影期間夠用，不需跨 session 持久化）|
| 展示情境 | ① 問青安利率（RAG）② 問自己資格（session context）③ 查案件進度（applicationId）|

### 實作架構

```
用戶輸入任意問題
    ↓
buildAiConsultingResponse(session, message)
    ↓
System prompt 組裝：
  [客戶資料] session.basicInfo + applicationId
  [本次對話] session.chatHistory 最近 5 輪
  [知識庫] ragQuery(message) 結果
    ↓
呼叫 claude-sonnet-4-6
    ↓
append 到 chatHistory → 回傳回覆
```

### 需修改檔案

| 檔案 | 異動 |
|------|------|
| `models/enums.ts` | 新增 `AI_CONSULTING` 狀態 |
| `models/types.ts` | UserSession 加 `chatHistory` 欄位 |
| `core/conversationStateMachine.ts` | Rich Menu 第 6 格改為 AI 諮詢 |
| `services/conversationHandler.ts` | 全域指令 + 狀態處理 |
| `services/aiConsultingService.ts` | 新建：prompt 組裝 + LLM 呼叫 |

| `services/conversationHandler.ts` | 全域觸發「AI諮詢」+ AI_CONSULTING 狀態處理 |

### 架構（RAG + Memory Augmentation）

```
用戶輸入問題
    ↓
system prompt 組裝：
  [客戶資料] session.basicInfo + applicationId
  [對話歷史] chatHistory 最近 5 輪
  [知識庫]   ragQuery(message) 結果
    ↓
claude-sonnet-4-6 生成回覆
    ↓
append 至 chatHistory → 回傳
```

### Demo 錄影腳本
1. 點 💬 AI 諮詢 → 歡迎 Flex
2. 問「青安利率多少？」→ RAG 引用知識庫回答
3. 問「我 35 歲公務員符合嗎？」→ 帶 session context 個人化回答
4. 問「我的案件到哪了？」→ 帶 applicationId 回應
5. 「返回主選單」退出
